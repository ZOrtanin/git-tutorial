### Конспект по Git: Основные команды и концепции

---

#### **Основные команды Git**

**Настройка Git:**

Установить имя пользователя и email:

```bash    
git config --global user.name "Ваше Имя" 
git config --global user.email "ваш_email@example.com" 
```


**Создание репозитория:**

Инициализация репозитория:
    
```bash
git init
```



**Работа с файлами:**

Просмотр статуса файлов:

```bash
git status
```

Добавление файлов в индекс (staging area):

```bash
git add <имя_файла>
```

Фиксация изменений (коммит):

```bash
git commit -m "Сообщение коммита"
```



**Просмотр истории коммитов:**

Просмотр истории:

```bash
git log
```

Просмотр детальной информации о коммите:

```bash
git show <хэш_коммита>
```



**Отмена изменений:**

Отмена изменений в файле до последнего коммита:

```bash
git restore <имя_файла>
```

Отмена изменений в индексе:

```bash
git restore --staged <имя_файла>
```



**Работа с ветками:**

Создание новой ветки:

```bash
git branch <имя_ветки>
```

Переключение на ветку:

```bash
git checkout <имя_ветки>
```

Слияние веток:

```bash
git merge <имя_ветки>
```

Удаление ветки:

```bash
git branch -d <имя_ветки>
```



**Работа с удалёнными репозиториями:**

Добавление удалённого репозитория:

```bash
git remote add origin <URL_репозитория>
```

Отправка изменений на удалённый репозиторий:

```bash
git push origin <имя_ветки>
```

Получение изменений с удалённого репозитория:

```bash
git pull origin <имя_ветки>
```



**Работа с тегами:**

Создание тега:

```bash
git tag <имя_тега>
```

Отправка тегов на удалённый репозиторий:

```bash
git push origin --tags
```

#### **Основные концепции Git**

1. **Что такое Git?**Git — это распределённая система контроля версий (VCS), которая позволяет фиксировать изменения в файлах и возвращаться к предыдущим версиям при необходимости.
    
2. **Состояния файлов в Git:**
    
    - **Untracked**— файл не отслеживается Git.
    - **Tracked**— файл отслеживается Git.
    - **Modified**— файл изменён, но не добавлен в индекс.
    - **Staged**— файл добавлен в индекс и готов к коммиту.
    - **Committed**— изменения зафиксированы в репозитории.
3. **Ветвление в Git:**Ветки позволяют работать над разными задачами параллельно. Основная ветка обычно называется`master`или`main`.
    
4. **Слияние (Merge) и Перебазирование (Rebase):**
    
    - **Merge**— объединяет изменения из одной ветки в другую, создавая коммит слияния.
    - **Rebase**— переносит коммиты из одной ветки на вершину другой, создавая линейную историю.
5. **Работа с удалёнными репозиториями:**
    
    - **Push**— отправляет изменения на удалённый репозиторий.
    - **Pull**— получает изменения с удалённого репозитория и сливает их с локальной веткой.
    - **Fetch**— получает изменения с удалённого репозитория, но не сливает их.
6. **Разрешение конфликтов:**Конфликты возникают, когда Git не может автоматически объединить изменения. Их необходимо разрешать вручную.
    

---

#### **Рекомендации по использованию Git**

1. **Работайте в отдельных ветках**для каждого нового функционала или исправления.
2. **Делайте коммиты часто**и с понятными сообщениями.
3. **Старайтесь поддерживать линейную историю**коммитов, используя`rebase`вместо`merge`, где это возможно.
4. **Используйте`.gitignore`**, чтобы исключать ненужные файлы из репозитория.
5. **Не работайте напрямую в ветке`master`**— используйте её только для стабильного кода.
